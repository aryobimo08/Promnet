git adalah:

GIT adalah sebuah version control system yang telah digunakan oleh para developer untuk dapat mengembangkan software secara bersamaan. Fungsi utama dari GIT adalah mengatur versi source code program dengan memberikan tanda baris serta code mana yang perlu ditambah ataupun diganti.

Ada beberapa perintah dasar dalam GIT, di antaranya adalah:

  1. GIT init: untuk membuat sebuah repository yang berada pada file lokal di mana nantinya file tersebut bernama .git.
  2. Git status: untuk mengetahui status dari sebuah repository lokal.
  3. GIT add: untuk menambahkan file baru yang ada pada repository yang telah dipilih.
  4. GIT commit: untuk menyimpan perubahan yang telah dilakukan.
  5. Git branch: untuk melihat seluruh branch yang ada dalam repository.
  6. GIT checkout: untuk dapat menukar branch yang aktif dengan branch yang telah dipilih.
  7. GIT merge: untuk menggabungkan branch yang aktif dan yang telah dipilih.
  8. GIT clone: untuk dapat membuat Salinan repository lokal.

Perbedaan GIThub dan GIT

Perbedaan yang utama antara Git vs Github adalah pada bentuk pengoperasian dan fungsinya. Keduanya sama-sama memudahkan penggabungan dan pembagian kode, namun bedanya Git merupakan open source software yang dirancang untuk merangkum riwayat sumber dari coding. Sedangkan Github merupakan layanan hosting berbasis web sebagai repositori Git. Github juga menawarkan seluruh DVCS SCM yang diperkaya dengan berbagai fitur seperti manajemen tiket, manajemen proyek, hingga pelacakan bug. Melalui Github developer bias membagikan, mengakses, dan menyimpan salinan repositori dari jarak jauh.
